# Payload Admin Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Payload admin usable by non-technical Chinese-speaking editors and brand-match it to the public site (light theme, grouped nav, fully localized fields, friendly editor, custom dashboard, branded login, live preview).

**Architecture:** Almost all changes are declarative Payload config (`src/payload.config.ts` + collection/global files), one admin stylesheet (`src/app/(payload)/custom.scss`), and a handful of React components under `src/components/admin/`. Live preview adds `@payloadcms/live-preview-react`, refactors the public News detail body into a client component driven by `useLivePreview`, and adds a new public `/activities/[slug]` page.

**Tech Stack:** Payload 3.85.0 · Next.js 16 (App Router) · React 19 · `@payloadcms/richtext-lexical` 3.85.0 · Postgres · pnpm 10.

## Global Constraints

- **Package manager is pnpm.** Commands: `pnpm build`, `pnpm generate:types`, `pnpm generate:importmap`, `pnpm dev`.
- **The admin layout `src/app/(payload)/layout.tsx` is auto-generated by Payload — DO NOT EDIT it.** All admin styling/fonts go through `src/app/(payload)/custom.scss` (already imported by that layout).
- **Brand colours:** vermillion `#8b2323`, deep `#6e1a1a`, paper `#f5f0e8`, paper-dark `#ede4d3`, warm-white `#fffdf8`, border `#e0d5bf`, ink `#1a1a1a`, ink-soft `#2c2c2c`, gold `#c5a55a`, stone `#8a857a`. Logo (會徽): `/images/logo.jpg` (in `public/`).
- **Override the `--theme-*` runtime variables, never `--color-*`.** Payload's primary/brand accent is the `--theme-success-*` ramp.
- **Regenerate the import map (`pnpm generate:importmap`) after adding/removing ANY custom component path in config**, or the admin throws "component not found".
- **Component paths in config are strings relative to `importMap.baseDir` = `src/`** (e.g. `/components/admin/Logo` → `src/components/admin/Logo.tsx`).
- **All editor-facing copy is Traditional Chinese (繁體, zh-HK/zh-TW).**
- **No test runner exists** and this work is config/CSS/RSC. Each task's "test" cycle = define the expected observable result, implement, then **verify** with build/type/importmap commands (expected: no errors) **plus a specific manual check** in `pnpm dev`, then commit.
- **DB schema:** the Postgres adapter has `push: true` in non-production, so new columns sync automatically when `pnpm dev` (or build) runs locally. No manual migration in dev.
- **Every commit message ends with this trailer** (shown in full in Task 1; abbreviated as `<trailer>` afterward):
  ```
  Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>
  ```
- Work happens on branch `admin-redesign` (already created; spec already committed).

import { safePayloadQuery } from "@/lib/payload";

type Item = { id: string | number; title: string; href: string; meta: string };

async function load() {
  return safePayloadQuery(
    async (payload) => {
      const [newsCount, actCount, memberCount, unreadCount, recentNews, recentAct, pending] =
        await Promise.all([
          payload.count({ collection: "news" }),
          payload.count({ collection: "activities" }),
          payload.count({ collection: "committee-members" }),
          payload.count({ collection: "enquiries", where: { readStatus: { not_equals: true } } }),
          payload.find({ collection: "news", sort: "-updatedAt", limit: 5 }),
          payload.find({ collection: "activities", sort: "-updatedAt", limit: 5 }),
          payload.find({
            collection: "enquiries",
            where: { readStatus: { not_equals: true } },
            sort: "-createdAt",
            limit: 5,
          }),
        ]);

      const recent: Item[] = [
        ...recentNews.docs.map((d) => ({
          id: `n${d.id}`,
          title: d.title,
          href: `/admin/collections/news/${d.id}`,
          meta: `消息 · ${fmt(d.updatedAt)}`,
        })),
        ...recentAct.docs.map((d) => ({
          id: `a${d.id}`,
          title: d.title,
          href: `/admin/collections/activities/${d.id}`,
          meta: `活動 · ${fmt(d.updatedAt)}`,
        })),
      ]
        .sort((x, y) => (x.id < y.id ? 1 : -1))
        .slice(0, 6);

      const pendingItems: Item[] = pending.docs.map((d) => ({
        id: d.id,
        title: d.name ?? "（無姓名）",
        href: `/admin/collections/enquiries/${d.id}`,
        meta: fmt(d.createdAt),

      return {
        stats: {
          unread: unreadCount.totalDocs,
          news: newsCount.totalDocs,
          activities: actCount.totalDocs,
          members: memberCount.totalDocs,

        pending: pendingItems,
